# Technical Writing Portfolio

Willkommen in meinem Technical-Writing-Portfolio.  
Dieses Repository enthält ausgewählte Arbeitsproben aus verschiedenen Bereichen der technischen Dokumentation.  
Die Beispiele sind so aufgebaut, dass sie typische Aufgaben aus der Praxis abdecken – von API-Dokumentation über Release Notes bis hin zu Nutzerhandbüchern.

## Inhalt

### 1. API-Dokumentation
**Ordner:** `/api-docs`  
Enthält eine Beispiel-API („Simple Weather API“) inklusive:
- Überblick  
- Endpunkte  
- Parameter- und Response-Spezifikationen  
- Fehlercodes  
- Beispielanfragen  

### 2. Release Notes
**Ordner:** `/release-notes`  
Strukturiertes Beispiel für ein Release (Version 1.2), inklusive:
- Neue Funktionen  
- Verbesserungen  
- Bugfixes  
- bekannte Einschränkungen  

### 3. Mini-Styleguide
**Ordner:** `/styleguide`  
Ein kompakter Styleguide für konsistentes Schreiben:
- Terminologieregeln  
- Formatierung  
- Tonalität  
- Beispielkonventionen  

### 4. Glossar
**Ordner:** `/glossar`  
Ein kleines terminologisches Glossar, das Begriffe aus den Beispieldokumenten definiert.

### 5. Testplan & Testfälle
**Ordner:** `/testing`  
Ein Beispiel für strukturiertes Testdokument:
- Teststrategie  
- Testumfang  
- Testfälle  
- erwartete Ergebnisse  

### 6. Nutzerhandbuch
**Ordner:** `/user-guide`  
Kurzanleitung für eine fiktive Desktop-App, inkl.:
- Installation  
- erste Schritte  
- Screens (schematisch)  
- Troubleshooting  

---

## Ziel des Portfolios
Dieses Portfolio soll demonstrieren:

- klar strukturierte technische Kommunikation  
- konsistente Verwendung von Terminologie  
- sachliche, präzise Sprache  
- technische Komplexität verständlich heruntergebrochen  
- differenzierte Dokumentationsformen (API, User Guide, Release Notes, Tests)
